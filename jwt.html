JWT

Zawartość tokena JWT w projekcie

Część nagłówkowa 

{
  "alg": "HS256",
  "typ": "JWT"
}

Zawartość właściwa tokena 

{
  "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier": "Maciej",
  "http://schemas.microsoft.com/ws/2008/06/identity/claims/role": "Administrator",
  "exp": 1675485682,
  "iss": "https://localhost:44368/",
  "aud": "https://localhost:44368/"
}

Claims

Oprócz niezbędnych do weryfikacji tokena informacji oraz czasu wygaśnięcia tokena, treść właściwa zawiera również dwa claimy:
  • Nazwę użytkownika posiadacza tokena
  • Rola uprawnień posiadacza tokena

Zmienna exp

Token zawiera również zmienną exp. Jest to czas wygaśnięcia tokena. Zawsze czas ważności tokena w projekcie wynosi 8 godzin. Oznacza to, że aby utrzymać sesję frontend, musi użyć standardowej procedury logowania.
